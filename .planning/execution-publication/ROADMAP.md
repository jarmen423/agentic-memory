# Publication Wave Roadmap

## Active Track: `w15-public-plugin-publication`

### Wave 0: Orchestrator lock

- `W15-PUB-00`: seed a standalone publication execution registry, freeze task
  ids, freeze ownership boundaries, and bind the registry to
  `docs/PLUGIN_GA_PLAN.md`.

### Wave 1: Contract lock

- `W15-PUB-01`: lock the public MCP contract and reviewer-facing backend
  requirements before any submission package work begins.

### Wave 2: Parallel submission-prep threads

- `W15-PUB-02`: OpenAI submission package, reviewer runbook, and publish checks.
- `W15-PUB-03`: Codex local/preflight plugin bundle, install checklist, and
  future self-serve readiness artifacts.
- `W15-PUB-04`: Anthropic remote connector submission package, examples, and
  reviewer runbook.
- `W15-PUB-05`: shared legal/support/runbook assets and launch-ops materials
  that both platform submissions depend on.

### Wave 3: Integration gate

- `W15-PUB-06`: reconcile the parallel submission assets into top-level product
  docs and the shared publication index without reopening the contract-lock
  files.

### Wave 4: Approval and launch gate

- `W15-PUB-07`: run the publication dry-run, track external review state,
  capture approval evidence, and close the launch gate once OpenAI and
  Anthropic publication paths are both satisfied.

### Wave 5: Verification

- run backend contract and auth regression gates
- validate Codex plugin manifest and MCP bundle JSON
- verify publication docs cross-links and reviewer packets
- update registry task status and handoffs with exact gate evidence
