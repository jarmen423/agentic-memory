# W15-PUB-01 Handoff

## Status

- Completed

## What Changed

- Added a frozen public-tool annotation contract in
  `src/am_server/mcp_profiles.py`.
- Applied that contract to the hosted public MCP surface in
  `src/agentic_memory/server/public_mcp.py`.
- Extended `register_conversation_tools(...)` in
  `src/agentic_memory/server/tools.py` so the shared conversation tools can be
  registered with explicit annotations on the public surface without changing
  the internal registration path.
- Added strict hosted-MCP auth mode in `src/am_server/auth.py` via
  `AM_SERVER_STRICT_MCP_AUTH=1` so hosted deployments can require dedicated
  surface keys instead of silently falling back to the REST API key.
- Added browser-preflight CORS handling in `src/am_server/app.py`:
  - default allowed origins for ChatGPT/OpenAI and Claude surfaces
  - `AM_SERVER_CORS_ALLOW_ORIGINS` override
  - MCP auth middleware skips `OPTIONS` preflight requests
- Added authenticated MCP smoke inventory endpoint in
  `src/am_server/routes/health.py`:
  - `GET /health/mcp-surfaces`
  - reports mounted surfaces, expected vs actual tool lists, strict-auth mode,
    CORS allowlist, and public annotation coverage
- Added unit coverage in `tests/test_server.py` for the frozen public contract.
- Extended `tests/test_am_server.py` to cover:
  - strict MCP auth mode
  - MCP surface health inventory
  - browser preflight behavior for `/mcp`

## Contract Decisions Locked

- Public read tools are annotated with:
  - `readOnlyHint=true`
  - `destructiveHint=false`
  - `idempotentHint=true`
  - `openWorldHint=false`
- Public write tools (`memory_ingest_research`, `add_message`) are annotated
  with:
  - `readOnlyHint=false`
  - `destructiveHint=true`
  - `idempotentHint=false`
  - `openWorldHint=false`
- Hosted MCP strict mode is opt-in through env so existing fallback behavior is
  preserved unless the deployment explicitly enables strict surface keys.

## Residual Risks / Follow-Up

- The new strict auth mode and CORS env var are implemented in code but are not
  yet documented in top-level product docs; that belongs in the later docs and
  integration tasks.
- The smoke endpoint uses REST auth rather than MCP-surface auth because it is
  intended as an operator endpoint, not a public client endpoint.
- Internal/full MCP tools are not yet given the same annotation treatment. That
  is intentional; this task locked the **public** publication contract only.
