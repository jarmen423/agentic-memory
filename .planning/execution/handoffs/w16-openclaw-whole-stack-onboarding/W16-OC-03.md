## Behavior Locked In This Task

- Temporal publish no longer relies on a saved `local` SpacetimeDB alias.
- Temporal bindings generation no longer relies on whatever CLI default target
  happens to be configured on the operator machine.
- `STDB_URI` is now required anywhere the temporal worker or sync worker needs
  to talk to SpacetimeDB.
- The supported local-stack story is now:
  - choose a real SpacetimeDB host/port
  - export `STDB_URI`
  - run the repo scripts against that explicit target
