**popup.html** — Single-page popup (~300px wide). Sections:
- Header: "am-ext" title
- Platform row: label "Platform:" + value span (e.g., "claude.ai" or "Not on a supported page")
- Status row: label "Status:" + colored badge span ("Capturing" green / "Paused" yellow / "Not configured" gray)
- Session row: label "Session:" + truncated session_id (show first 8 chars + "…") — populated via chrome.tabs.sendMessage
- Turns row: label "Turns this session:" + count span
- Action row: single button with id="pause-btn" (text toggles: "Pause" / "Resume")
- Footer link: "Settings" → opens onboarding.html in new tab

No external CSS frameworks. Inline styles or popup.css for layout. Keep minimal — functional over beautiful.

**popup.js** — On DOMContentLoaded:
1. Query active tab: `chrome.tabs.query({active: true, currentWindow: true}, tabs => { ... })`
2. Detect platform from tab.url hostname (same logic as content.js detect_platform)
3. Read config: `chrome.storage.sync.get(["endpoint", "api_key", "paused", "platforms"], cfg => { ... })`
4. Determine status:
   - If !cfg.api_key → "Not configured" (gray)
   - Else if cfg.paused || (platform && cfg.platforms?.[platform] === false) → "Paused" (yellow)
   - Else if platform → "Capturing" (green)
   - Else → "Not on a supported page" (gray)
5. Set pause button text: cfg.paused ? "Resume" : "Pause"
6. Hide pause button if no platform (not on supported page)
7. Pause button onclick: toggle cfg.paused, write back via chrome.storage.sync.set({paused: !cfg.paused}), update button text and status badge immediately (no reload needed)
8. Settings link: `chrome.tabs.create({url: chrome.runtime.getURL("onboarding/onboarding.html")})`

Turn count and session ID: attempt chrome.tabs.sendMessage to active tab requesting {type: "GET_STATUS"}. If fails (tab not a supported page), leave those fields as "--". Note: content.js in Plan 02 does NOT implement GET_STATUS handler — this is a best-effort display. If the response is absent, silently show "--". Do NOT block popup render waiting for this.

Use snake_case for all JS variable/function names per project convention.
  </action>

**onboarding.html** — Full-page configuration form. Sections:
- Header: "am-ext Setup" title + subtitle "Configure your agentic memory connection"
- am-server URL field: label "Memory Server URL", text input id="endpoint", placeholder "http://localhost:8000"
- API Key field: label "API Key", password input id="api-key", placeholder "Your Bearer token"
- Project ID field: label "Project ID", text input id="project-id", placeholder "browser"
- Test Connection button id="test-btn" + status indicator span id="test-status" (hidden initially)
- Section "Capture platforms" with 4 checkboxes: id="cb-chatgpt", "cb-claude", "cb-gemini", "cb-perplexity" — all checked by default
- Save button id="save-btn"
- Success message div id="saved-msg" (hidden initially): "Settings saved."

**onboarding.js** — On DOMContentLoaded:
1. Load existing config from chrome.storage.sync and pre-fill all form fields
2. Test Connection button onclick:
   - Disable button during test
   - fetch(`${endpoint_input.value || "http://localhost:8000"}/health`, {signal: AbortSignal.timeout(5000)})
   - On success (res.ok): show test-status as green "Connected"
   - On failure (network error or !res.ok): show test-status as red "Cannot reach server"
   - Re-enable button after test
3. Save button onclick:
   - Read all form values
   - Build config object matching chrome.storage.sync schema exactly:
     ```js
     const config = {
       endpoint: endpoint_input.value.trim() || "http://localhost:8000",
       api_key: api_key_input.value.trim(),
       project_id: project_id_input.value.trim() || "browser",
       paused: false,
       platforms: {
         chatgpt: cb_chatgpt.checked,
         claude: cb_claude.checked,
         gemini: cb_gemini.checked,
         perplexity: cb_perplexity.checked,
       },
     };
     ```
   - chrome.storage.sync.set(config, () => { show saved-msg for 2 seconds })
4. All fetch errors silent — test button catches and shows red indicator, never throws

Use AbortSignal.timeout(5000) for the health check (Chrome 103+, available in extension context).
Keep form functional on first load even if chrome.storage.sync.get returns empty (use defaults).
