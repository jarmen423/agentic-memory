  <action>
    Add to `src/codememory/core/graph_writer.py` after the existing `write_relationship()` method:

    **`write_temporal_relationship()`**

    MERGE pattern MUST match on relationship type + endpoint node identity ONLY. No temporal
    fields in the MERGE clause itself (this would create duplicate relationships on re-ingest).

    Correct Cypher template (adapt from RESEARCH.md Pattern 1):
    ```python
    cypher = (
        "MATCH (m {source_key: $source_key, content_hash: $content_hash})\n"
        "MATCH (e {name: $entity_name, type: $entity_type})\n"
        f"MERGE (m)-[r:{rel_type}]->(e)\n"
        "ON CREATE SET r.valid_from = $valid_from,\n"
        "              r.valid_to = $valid_to,\n"
        "              r.confidence = $confidence,\n"
        "              r.support_count = $support_count,\n"
        "              r.contradiction_count = $contradiction_count\n"
        "ON MATCH SET  r.support_count = r.support_count + 1,\n"
        "              r.confidence = CASE WHEN $confidence > r.confidence\n"
        "                                  THEN $confidence\n"
        "                                  ELSE r.confidence END"
    )
    ```

    Parameters: `source_key`, `content_hash`, `entity_name`, `entity_type`, `rel_type`,
    `valid_from` (str, required), `valid_to` (str | None = None), `confidence` (float = 1.0),
    `support_count` (int = 1), `contradiction_count` (int = 0).

    **`update_relationship_validity()`**

    MATCH the specific relationship by source_key + content_hash + entity name/type + rel_type,
    then SET r.valid_to = $valid_to. Use MATCH (not MERGE) — this is an update, not upsert.

    Parameters: `source_key`, `content_hash`, `entity_name`, `entity_type`, `rel_type`, `valid_to` (str).

    **`increment_contradiction()`**

    MATCH the same pattern, SET r.contradiction_count = r.contradiction_count + 1.
    Does NOT change valid_to — flagging only.

    Parameters: `source_key`, `content_hash`, `entity_name`, `entity_type`, `rel_type`.

    **Also update the five dedicated methods** to accept temporal keyword args with defaults:
    - `write_has_chunk_relationship`: add `valid_from: str | None = None, confidence: float = 1.0`
    - `write_part_of_relationship`: same
    - `write_cites_relationship`: same
    - `write_has_turn_relationship`: same
    - `write_part_of_turn_relationship`: same

    When `valid_from` is None, default to `datetime.now(timezone.utc).isoformat()`.
    Add `from datetime import datetime, timezone` import at top of file if not present.

    In each dedicated method's ON CREATE SET clause, add the five temporal fields:
    `r.valid_from`, `r.valid_to = null`, `r.confidence`, `r.support_count = 1`,
    `r.contradiction_count = 0`.
    In ON MATCH SET, add `r.support_count = r.support_count + 1` (same ON MATCH increment
    pattern as write_temporal_relationship).
