Claim dict schema (locked in CONTEXT.md decision D):
```python
{
    "subject": str,        # entity name
    "predicate": str,      # one of the closed predicate catalog
    "object": str,         # entity name
    "valid_from": str | None,
    "valid_to": str | None,
    "confidence": float,   # 0.0–1.0
}
```

Closed predicate catalog (locked in CONTEXT.md decision E):
KNOWS, WORKS_AT, RESEARCHED, REFERENCES, USES, LEADS, PART_OF, LOCATED_IN, CREATED_BY, CONTRADICTS
"Other" predicate fallback = REFERENCES

write_temporal_relationship (added in 07-01):
```python
def write_temporal_relationship(
    self,
    source_key: str,
    content_hash: str,
    entity_name: str,
    entity_type: str,
    rel_type: str = "ABOUT",
    valid_from: str | None = None,
    valid_to: str | None = None,
    confidence: float = 1.0,
    support_count: int = 1,
    contradiction_count: int = 0,
) -> None: ...
```

NOTE: For Entity-to-Entity claim relationships, the "source" is the subject entity (not a Memory
node). The graph_writer MERGE pattern matches on `{source_key, content_hash}` for Memory nodes.
For Entity-to-Entity writes, construct a dedicated Cypher MATCH in the pipeline directly
(do not pass Memory node keys). See action section for exact Cypher pattern.

    2. In `__init__`, instantiate claim extraction service alongside the existing NER service:
       ```python
       self._claim_extractor = ClaimExtractionService(
           api_key=groq_api_key,
           model=groq_model,
           predicates=config.get("extraction", {}).get("claim_predicates"),
       ```

    3. In `_ingest_report()`, after the existing NER block (which calls
       `self._entity_extractor.extract()` and writes ABOUT/MENTIONS relationships), add a
       SEPARATE try/except block for claim extraction:

       ```python
       # Claim extraction pass — independent of NER (per CONTEXT.md decision F)
       try:
           claims = self._claim_extractor.extract(document_text)
           for claim in claims:
               self._write_claim(claim)
       except Exception as exc:
           logger.error("Claim extraction failed (non-blocking): %s", exc)
       ```

    4. Add `_write_claim(self, claim: dict[str, Any]) -> None` private method:

       This method writes an Entity-to-Entity relationship directly via a Neo4j session.
       It must NOT use write_temporal_relationship() which is designed for Memory→Entity writes.
       Instead, write a dedicated Cypher statement:

       ```python
       cypher = (
           "MERGE (subj:Entity {name: $subject_name})\n"
           "ON CREATE SET subj.type = $subject_type\n"
           "MERGE (obj:Entity {name: $object_name})\n"
           "ON CREATE SET obj.type = $object_type\n"
           f"MERGE (subj)-[r:{claim['predicate']}]->(obj)\n"
           "ON CREATE SET r.valid_from = $valid_from,\n"
           "              r.valid_to = $valid_to,\n"
           "              r.confidence = $confidence,\n"
           "              r.support_count = 1,\n"
           "              r.contradiction_count = 0\n"
           "ON MATCH SET  r.support_count = r.support_count + 1,\n"
           "              r.confidence = CASE WHEN $confidence > r.confidence\n"
           "                                  THEN $confidence\n"
           "                                  ELSE r.confidence END"
       )
       ```

       Use the claim's `valid_from` if not None, else `self._now()`.
       Use the claim's `confidence` directly (LLM-reported).

       CRITICAL: MERGE on relationship type + endpoint identity only (no temporal fields in MERGE
       clause). This is the same pattern as write_temporal_relationship().

       Subject/object entity types default to "unknown" when not determinable from the claim dict.
       (The claim dict has name but no type — use "unknown" as default Entity type for claim nodes.)

    5. Only call `_ingest_report` claim extraction pass, NOT `_ingest_finding`. Findings are
       typically shorter and less claim-rich; adding claims to findings is a future enhancement.
  </action>
