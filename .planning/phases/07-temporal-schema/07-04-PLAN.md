```python
from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.jobstores.sqlalchemy import SQLAlchemyJobStore
from apscheduler.triggers.cron import CronTrigger

SCHEDULES_DB = Path("~/.config/agentic-memory/schedules.db").expanduser()
jobstores = {"default": SQLAlchemyJobStore(url=f"sqlite:///{SCHEDULES_DB}")}
scheduler = BackgroundScheduler(jobstores=jobstores, daemon=True)
scheduler.start()

scheduler.add_job(
    func=run_research_job,
    trigger=CronTrigger.from_crontab("0 9 * * 1"),
    id=schedule_id,
    replace_existing=True,
    kwargs={"schedule_id": schedule_id},
)
```

  <action>
    Create `src/codememory/core/scheduler.py`.

    **Module-level constants:**
    ```python
    SCHEDULES_DB = Path("~/.config/agentic-memory/schedules.db").expanduser()
    MAX_RUNS_PER_DAY_DEFAULT = 5
    BRAVE_CIRCUIT_RESET_SECONDS = 3600  # 1 hour
    BRAVE_FAILURE_THRESHOLD = 3
    ```

    **ResearchScheduler class:**

    `__init__(self, connection_manager, groq_api_key, groq_model, brave_api_key, pipeline)`:
    - `connection_manager`: ConnectionManager instance (for Neo4j reads/writes)
    - `groq_api_key`, `groq_model`: for LLM variable fill calls
    - `brave_api_key`: for Brave Search HTTP calls
    - `pipeline`: ResearchIngestionPipeline instance (injected, not constructed internally)
    - Initialize APScheduler BackgroundScheduler with SQLite job store (create DB path dir if needed)
    - Initialize Groq client for variable fill calls
    - Initialize circuit breaker state: `_brave_failures = 0`, `_brave_circuit_open_until: float = 0.0`
    - Call `scheduler.start()` during init

    `create_schedule(self, template, variables, cron_expr, project_id, max_runs_per_day=MAX_RUNS_PER_DAY_DEFAULT) -> str`:
    - Generate `schedule_id = str(uuid.uuid4())`
    - Write :Schedule node to Neo4j via connection_manager session:
      ```cypher
      MERGE (s:Schedule {schedule_id: $schedule_id})
      SET s.template = $template, s.variables = $variables,
          s.cron_expr = $cron_expr, s.project_id = $project_id,
          s.created_at = $created_at, s.last_run_at = null,
          s.run_count = 0, s.max_runs_per_day = $max_runs_per_day
      ```
    - Register APScheduler job:
      ```python
      self._scheduler.add_job(
          func=_sync_run_research_job,
          trigger=CronTrigger.from_crontab(cron_expr),
          id=schedule_id,
          replace_existing=True,
          kwargs={"scheduler": self, "schedule_id": schedule_id},
      )
      ```
    - Return schedule_id

    `run_research_session(self, schedule_id, ad_hoc_template=None, ad_hoc_variables=None) -> dict`:
    - Load schedule from Neo4j (or use ad_hoc params if no schedule_id)
    - **Cost cap check:** if schedule's run_count >= max_runs_per_day, log warning and return early
    - **Brave circuit breaker check:** if `time.time() < self._brave_circuit_open_until`, log and
      return early
    - Call `_fill_variables(template, variables, project_id)` to get filled query string
    - Call Brave Search via httpx. On non-2xx: increment `_brave_failures`; if >= threshold,
      set `_brave_circuit_open_until = time.time() + BRAVE_CIRCUIT_RESET_SECONDS`; return error dict
    - On 2xx: reset `_brave_failures = 0`
    - For each Brave result, call `self._pipeline.ingest(...)` with type="finding"
    - Write `RESEARCHED` edge from the project to topic entity via `write_temporal_relationship`
    - Update Schedule node: increment run_count, set last_run_at = now()
    - Return `{"status": "ok", "results": N, "query": filled_query}`

    `list_schedules(self, project_id) -> list[dict]`:
    - MATCH (:Schedule {project_id: $project_id}) RETURN s, order by created_at

    `_fill_variables(self, template, variables, project_id) -> str` (private):
    - Query Neo4j for last 10 RESEARCHED edges for this project:
      ```cypher
      MATCH ()-[r:RESEARCHED]->(e)
      WHERE r.project_id = $project_id
      RETURN e.name AS topic, r.valid_from AS researched_at
      ORDER BY r.valid_from DESC LIMIT 10
      ```
    - Build Groq prompt with template, variable names, recent topics list, instruction to avoid
      already-covered topics
    - Call Groq with `response_format={"type": "json_object"}`, `temperature=0.7` (creative
      diversity for variable selection)
    - Parse returned JSON dict, substitute into template, return filled string
    - On Groq failure: log error, substitute empty strings for variables, return partially filled
      template (do not raise — scheduler must be resilient)

    **Sync wrapper for APScheduler** (module-level function, not a method):
    ```python
    def _sync_run_research_job(scheduler: "ResearchScheduler", schedule_id: str) -> None:
        """Sync wrapper called by APScheduler in background thread."""
        import asyncio
        asyncio.run(scheduler._async_run_research_session(schedule_id))
    ```
    Note: `run_research_session()` should be sync (httpx sync client). Only make it async if
    the pipeline's `ingest()` is async. Check web/pipeline.py — if `ingest()` is sync, keep
    the entire run_research_session sync and call it directly without asyncio.run().

    **Full type hints and Google docstrings on all public methods.**
