<objective>
Harden the code graph so it is safe to use for graph-aware retrieval, then add a repo-scoped
non-temporal code PPR path behind a feature flag.
</objective>

<must_haves>

- `repo_id` is part of the code-domain identity model.
- The canonical parser is the main extraction contract for Python and JS/TS-like code.
- Fuzzy import edge creation is removed from the primary retrieval graph.
- Code search accepts optional `repo_id` and remains backward compatible when omitted.
- Code PPR is non-temporal, uses only high-confidence edges, and stays behind `ENABLE_CODE_PPR`.

</must_haves>
