<objective>
Make `CALLS` trustworthy enough to be a real cross-language feature for Python,
JavaScript, and TypeScript by fixing the generic analyzer-to-graph handoff.

Purpose: live repo testing already proved that the product is usable without
`CALLS`, but it also proved that the current semantic path is too brittle to be
considered shippable. The next step is not to customize `m26pipeline`; it is to
turn semantic call analysis into a reusable, diagnosable subsystem.

Output: a Python semantic analyzer path, hardened JS/TS mapping, repo-level
drop diagnostics, and regression tests that prove semantic edges survive full
indexing for more than one repository shape.
</objective>

<interfaces>
Required analyzer contract:
```text
repo-relative file path
caller qualified_name
outgoing targets with:
  - repo-relative target path
  - unqualified target name
  - best-effort qualified_name_guess
  - optional container_name
  - optional definition line/column when the analyzer can provide them
per-file diagnostics:
  - analyzer availability
  - project loading notes
  - dropped-target counts by reason
```

Required graph diagnostics contract:
```text
repo_id
total_functions
functions_with_calls
total_call_edges
high_confidence_edges
files_with_analyzer_edges
sources[]
drop_reasons[]
```
</interfaces>
