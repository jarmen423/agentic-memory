# Wave 0 Handoff - Phase 11 Contract Lock

## Locked Decisions

- Code graph identity is repo-scoped via `repo_id`.
- `project_id` is not used as a surrogate repository identifier.
- Code PPR is non-temporal in v1.
- `ENABLE_CODE_PPR` gates the new reranker.
- v1 traversal edges are limited to `IMPORTS`, `DEFINES`, and `HAS_METHOD`.
- `CALLS` remains out of the traversal graph until its precision is benchmarked.

## Immediate Consequence

All downstream implementation should assume:

- code search functions may accept `repo_id`,
- ingestion must preserve repo scope,
- watcher updates cannot bypass the repo-scoped graph contract,
- broad fuzzy matching is not allowed in the primary retrieval graph.
