# OpenClaw Testing + Dashboard Wave

## Objective

Turn the post-foundation OpenClaw prototype into an operator-visible product
surface with a real dashboard and a deeper verification harness, without yet
expanding into packaging, marketplace submission, or GTM rollout.

## Workstreams

1. Planning and execution truth lock.
2. Backend dashboard API contract.
3. Dashboard workspace and desktop shell replacement.
4. Operational verification harnesses.
5. Integration gate: dashboard CI and cross-surface verification.

## Merge Gates

- `python -m pytest tests/test_am_server.py tests/test_openclaw_contract.py desktop_shell/tests/test_app.py -q`
- `python -m pytest tests/e2e/test_openclaw_e2e.py tests/load/test_openclaw_load.py tests/chaos/test_openclaw_chaos.py -q`
- `npm run build --workspace am-dashboard`
- `npm run test --workspace am-dashboard`
- `npm run typecheck --workspace am-dashboard`
