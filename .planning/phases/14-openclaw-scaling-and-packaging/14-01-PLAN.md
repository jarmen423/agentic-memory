# OpenClaw Scaling + Packaging Wave

## Objective

Take the now-functional OpenClaw stack from internal feature completeness to
beta-rollout readiness by hardening the backend scale path, making the plugin
package distribution-ready, and adding production deployment and release
artifacts.

## Workstreams

1. Planning and execution truth lock.
2. Backend throughput, MCP surface, and observability hardening.
3. `packages/am-openclaw/` packaging and install/distribution readiness.
4. Production deployment and release artifact scaffolding.
5. Integration gate: CI, packaging validation, and deployment config checks.

## Merge Gates

- `python -m pytest tests/test_am_server.py tests/test_openclaw_contract.py desktop_shell/tests/test_app.py -q`
- `python -m pytest tests/e2e/test_openclaw_e2e.py tests/load/test_openclaw_load.py tests/chaos/test_openclaw_chaos.py -q`
- `npm run build --workspace agentic-memory`
- `npm run test --workspace agentic-memory`
- `npm run typecheck --workspace agentic-memory`
- `npm run build --workspace am-dashboard`
- `npm pack --workspace agentic-memory --dry-run`
- `docker compose -f docker-compose.prod.yml config`
