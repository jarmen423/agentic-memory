# OpenClaw Docs + Private Beta Wave

## Objective

Turn the now-deployable OpenClaw stack into a partner-onboardable private beta
by locking the public install story, committing the backend OpenAPI contract,
preparing marketplace/publish surfaces, and writing the user, operator, and
support docs required for the first design-partner cohort.

## Workstreams

1. Planning and execution truth lock.
2. User/operator docs plus committed OpenAPI output.
3. Package identity, marketplace, and publish-surface preparation.
4. Private-beta onboarding, support, and partner-operations artifacts.
5. Integration gate: docs, package identity, release wiring, and regression validation.
