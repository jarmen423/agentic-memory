# OpenClaw Whole-Stack Onboarding Wave

## Objective

Turn the current OpenClaw private-beta-prepped stack into a whole-stack
onboarding flow that a real operator can complete without reverse-engineering
ports, saved CLI aliases, optional temporal services, or hidden local
dependencies.

## Workstreams

1. Planning and execution truth lock.
2. Onboarding contract lock across plugin, backend, shell, and local services.
3. Plugin doctor/setup UX.
4. Whole-stack bootstrap and temporal-target cleanup.
5. Docs and troubleshooting consolidation.
6. Integration gate and regression validation.
