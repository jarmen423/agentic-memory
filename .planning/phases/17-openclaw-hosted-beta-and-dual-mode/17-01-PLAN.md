# OpenClaw Hosted Beta Platform + Dual-Mode Wave

## Objective

Turn the current OpenClaw beta stack into a truthful managed hosted beta on the
existing GCP VM while keeping self-hosted as a supported full-stack fallback.

## Workstreams

1. Planning and execution truth lock.
2. Managed-vs-self-hosted contract lock across backend onboarding, auth, and operator boundaries.
3. Backend hosted-beta auth, workspace scoping, metering, and provisioning helpers.
4. Plugin setup/doctor UX for hosted vs self-hosted mode.
5. Managed-beta deployment docs and self-hosted runbooks.
6. Integration gate and regression validation.

## Merge Gates

- `python -m pytest tests/test_am_server.py tests/test_openclaw_contract.py -q`
- `npm run build:openclaw`
- `npm run test:openclaw`
- `npm run typecheck:openclaw`
