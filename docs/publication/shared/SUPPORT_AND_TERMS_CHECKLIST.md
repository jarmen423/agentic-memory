# Support and Terms Checklist

Use this when turning the shared policy requirements into actual publication-ready pages.

## Support page requirements

- Public entrypoint exists at a stable URL.
- Contact path is obvious and does not require account creation.
- Support scope covers:
  - ChatGPT/OpenAI app connection issues
  - Anthropic connector setup issues
  - demo/reviewer account issues
  - privacy and deletion requests
- Response expectation is stated in plain language.
- Page includes current product name and current publisher name.

## Support contact requirements

- One canonical support email or form is chosen.
- That same contact appears in:
  - OpenAI submission
  - Anthropic submission
  - product website/support page
  - future Codex listing metadata
- Contact owner is assigned internally.

## Terms page requirements

- Current product description is accurate.
- Acceptable-use expectations align with platform policies.
- Abuse, suspension, and removal rights are stated.
- Warranty/availability language is deliberate and not copied blindly from a template.
- Legal notice/contact path is present.

## Launch-readiness check

- Privacy, terms, support, and website URLs all live on the same stable domain family.
- None of the public URLs depend on a branch name, blob path, or ephemeral doc hosting.
- URLs are already live before either review submission is created.
- Current target URL family:
  - `https://api.agenticmemory.com/publication/agentic-memory`
  - `https://api.agenticmemory.com/publication/privacy`
  - `https://api.agenticmemory.com/publication/terms`
  - `https://api.agenticmemory.com/publication/support`
  - `https://api.agenticmemory.com/publication/dpa`
